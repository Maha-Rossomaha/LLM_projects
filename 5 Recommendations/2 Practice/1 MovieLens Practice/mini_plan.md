## Цель (зафиксируем)

**Для top-100 самых популярных фильмов** найти **top-10 самых похожих фильмов**
по **косинусной близости профилей оценок пользователей** (MovieLens 100k).

---

## Подзадачи по шагам

### 1. Загрузка и базовая подготовка данных

* Загрузить `ratings`:

  * `user_id`, `movie_id`, `rating`
* Проверить:

  * user_id и movie_id → привести к **0-based индексам** (если нужно)
* Решение:

  * что считать “популярностью” → **число оценок**.

---

### 2. Найти top-100 самых популярных фильмов

* Посчитать:

  * `cnt(movie_id) = number of ratings`
* Отсортировать по убыванию
* Взять **100 movie_id**
* Это **фиксированный поднабор**, дальше работаем только с ним.

👉 Важно: similarity считаем **между фильмами**, но профили строятся **по всем пользователям**.

---

### 3. Построить user–item матрицу (R)

* Матрица размера:
  $$
  R \in \mathbb{R}^{|U| \times |I|}
  $$
* Где:

  * строки — пользователи,
  * столбцы — фильмы,
  * значения — рейтинги (обычно 1–5).
* Использовать **sparse формат** (CSR).

💡 На этом шаге пока **не фильтруем до top-100**.

---

### 4. Извлечь item-профили для top-100

Для каждого фильма $i$ из top-100:

* взять столбец:
  $
  r_{:i}
  $
* это **вектор оценок по всем пользователям**.

На практике:

* либо работать с подматрицей $R_{:, I_{top100}}$,
* либо доставать столбцы по одному.

---

### 5. Определить косинусную близость

Для двух фильмов (i, j):

$$
\mathrm{sim}(i,j) = \frac{r_{:i}^\top r_{:j}}
{|r_{:i}| \cdot |r_{:j}|}
$$

Решения, которые тебе нужно принять:

* считать **по всем пользователям** (включая нули),
* или только по **пересечению пользователей с оценками**
  (обычно для учебного примера — по всем).

---

### 6. Посчитать item–item similarity для top-100

* Для каждого фильма $i$ из top-100:

  * посчитать similarity со **всеми остальными фильмами из top-100**.
* Получится матрица:
  $$
  S \in \mathbb{R}^{100 \times 100}
$$

Обязательные действия:

* `sim(i,i) = 0` (или `-inf`),
* убедиться, что матрица симметрична.

---

### 7. Найти top-10 похожих для каждого фильма

Для каждого фильма $i$:

* отсортировать $S_{i,:}$ по убыванию,
* взять **10 фильмов с максимальной similarity**,
* сохранить:

  * `movie_i → [(movie_j, sim_ij), …]`.

---

### 8. Проверки здравого смысла (очень важно)

* Похожие фильмы:

  * одной франшизы?
  * одного жанра?
* Нет ли:

  * одинаковых значений similarity для всего?
  * доминирования супер-популярных фильмов?

Если всё “слишком хорошо” — это повод подумать о:

* mean-centering,
* shrinkage,
* implicit vs explicit.

## Что будет следующим логичным шагом

После этого упражнения обычно идут:

1. **mean-centering** (убрать user bias),
2. **shrinkage** по числу общих пользователей,
3. сравнение с **EASE / SLIM**,
4. переход от “similar movies” к **рекомендациям пользователю**.
