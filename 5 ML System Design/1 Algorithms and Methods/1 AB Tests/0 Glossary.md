# A/B‑Testing Glossary

## Базовая статистика
- **Гипотезы ($H_0$/$H_1$)** — нулевая/альтернативная: «эффекта нет» vs «эффект есть».
- **Уровень значимости ($\alpha$)** — допустимая доля ложных находок (ошибок I рода), обычно 0.05.
- **Ошибка I рода (Type I, $\alpha$‑error)** — отвергли $H_0$, когда она верна (ложная тревога).
- **Ошибка II рода (Type II, $\beta$‑error)** — не отвергли $H_0$, когда эффект есть (пропустили).
- **Мощность (Power = 1−$\beta$)** — вероятность, что тест действительно обнаружит эффект, если эффект существует.
- **p‑value** — вероятность получить (наблюдать) не менее экстремальные данные при верной $H_0$. Т. е. если p-value слишком маленькое, то наблюдаемые данные слишком маловероятны (слишком странные) при верном $H_0$.
- **Доверительный интервал (CI)** — диапазон значений параметра, согласованных с данными (частотная интерпретация).
- **Кредитивный интервал (CrI)** — байесовский аналог CI: диапазон с заданной апостериорной вероятностью.
- **Практическая значимость** — бизнес‑вес эффекта (денежный/продуктовый), в отличие от чисто статистической.

## Эффект и метрики
- **Эффект (effect size)** — величина изменения:
  - абсолютный (п.п. для долей, единицы метрики для средних),
  - относительный (%),
  - стандартизованный (Cohen’s *d* для средних; Cohen’s *h* для долей).
- **MDE (Minimum Detectable Effect)** — минимальный эффект, который тест обязан «увидеть» при заданных $\alpha$ и power.
- **OEC (Overall Evaluation Criterion)** — главная метрика решения.
- **Guardrail‑метрики** — метрики‑ограничения, которые нельзя ухудшать.
- **CR/CTR** — конверсия/кликабельность (бинарные метрики).
- **ARPU/ARPPU** — средняя выручка на пользователя / плательщика (непрерывные метрики).

## Данные и распределения
- **Нормальность** — близость распределения к нормальному (для t‑тестов).
- **Скошенность (skewness)** — асимметрия распределения; правый хвост ⇒ положительная скошенность.
- **Куртозис (kurtosis)** — «тяжесть» хвостов относительно нормального.
- **Тяжёлые хвосты (heavy tails)** — повышенная вероятность экстремальных значений (часто в выручке/времени).
- **CLT (центральная предельная теорема)** — средние больших выборок ≈ нормальны.
- **Гомоскедастичность/гетероскедастичность** — равенство/неравенство дисперсий между группами.
- **Независимость наблюдений** — отсутствие зависимостей (важно для корректных SE и p‑value).

## Экспериментальный дизайн
- **Рандомизация (random assignment)** — случайное и независимое распределение в группы.
- **SRM (Sample Ratio Mismatch)** — несоответствие фактических и ожидаемых долей трафика.
- **Единица рандомизации / анализа** — кто рандомизируется и на чём считается метрика (user/session/show).
- **Стратификация / блокировка** — рандомизация внутри однородных слоёв для снижения дисперсии.
- **Кластеры и ICC** — зависимость внутри групп (регионы/корпорации); ICC → дизайн‑эффект и рост нужного N.
- **Контаминация** — «перетекание» воздействия между группами (нарушение изоляции).
- **Интерференция / SUTVA** — зависимость исходов пользователя от чужого лечения (нарушение допущения независимости).
- **ITT/TOT** — анализ «по назначению» (Intention‑to‑Treat) vs «по факту воздействия» (Treatment‑on‑the‑Treated).

## Снижение дисперсии и робастность
- **CUPED/CUPAC** — регрессионная корректировка метрик ковариатами (снижение $\sigma^2$ → меньше N).
- **Винзоризация / обрезка хвостов** — робастные процедуры для heavy‑tail.
- **Лог‑преобразование** — превращает мультипликативные эффекты в аддитивные, стабилизирует дисперсию.

## Планирование эксперимента
- **Power analysis** — расчёт N по $\alpha$, мощности и MDE.
- **Длительность** — N и доли трафика → оценка дней/недель (учитывать сезонность).
